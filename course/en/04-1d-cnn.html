Lesson 04

1D CNN: the network that sees temporal patterns 👀

Instead of memorizing the whole signal, learn a few small templates and slide them along time — and with this one idea, build the project's first real model.

The story 📖

Imagine you have a very long music tape and you are hunting for one specific bit of melody — say, the famous first four notes of a song. A silly approach would be to memorize the entire tape and compare all of it against your memory every single time. The smarter way: build a small "template" of that melody and slide it gently along the tape; wherever the template matches the tape, you make a mark.

Convolution is exactly that sliding of a template. A CNN (Convolutional Neural Network), instead of memorizing the whole signal, learns a large number of small templates — each one is called a kernel or filter — and uses them to find the important patterns anywhere in the signal. One template might, for instance, learn to recognize this pattern: "the steering wheel turned slightly while the distance to the lane line shrank". That is precisely the signature of the start of a lane change!

More precisely 🔬

In convolution, the template (the kernel) is a row of learnable numbers. The template slides along the signal and, at every position, multiplies its numbers with the slice underneath and adds them up. Wherever the shape of the signal resembles the shape of the template, the result gets large and forms a peak in the output. That output is called a feature map, or feature map:

Now, a few points that matter for our own data:

• Our data is 31 parallel signals over 50 time steps, so we use Conv1d: the kernel slides only along the time axis (unlike an image, which has two dimensions and calls for Conv2d).
• Each layer understands patterns slightly more complex than the layer before it: the first catches raw movements (like "the steering wheel turned"), the next one combinations ("the wheel turned and the distance shrank"), and the last the complete behavioral "signature" of a lane change.

And a golden trick for small models: depthwise-separable convolution. It is the ordinary convolution broken into two cheaper stages: first each channel is processed on its own with its own kernel, then the channels are combined with a kernel of size 1. The result: nearly the same accuracy with several times fewer parameters — perfect for a microcontroller.

Why "several times fewer"? Let's count the weights 🔢

Counting the weights of a convolution layer is easy — it's just a product of a few numbers:

• Ordinary Conv1d = input channels × output channels × kernel size. For example, 64 input channels, 64 output channels and a kernel of size 5 gives 64×64×5 = 20,480 weights.
• The separable version splits the same job in two: the depthwise stage (channels × kernel size = 64×5 = 320) and the 1×1 combination stage (input channels × output channels = 64×64 = 4,096). Total = 4,416 weights.

That is the same job with about 4.6 times fewer parameters — and the accuracy stays nearly the same.

At the bottom of the network sits a summarizer: Global Average Pooling takes just the mean of each channel — an aggressive but effective summary. A small linear layer then builds the final answer out of those few numbers.

The alternative is to flatten (Flatten) the entire feature map — that is, line up every time step and every channel side by side. Then the input of the next linear layer has size "window length × number of channels", which is huge and makes that layer heavy. GAP, however, shrinks the input to "just the number of channels", so the same decision layer becomes tens of times lighter — exactly the saving a microcontroller needs.

In our project 🧰

• Our baseline model — a small depthwise-separable 1D CNN — lives in src/models/baseline.py.
• Its structure is exactly this lesson: an input Conv1d (stem) + two depthwise-separable blocks + Global Average Pooling + one linear layer for the output.
• The entire training run took only 27 seconds!
